## 📖 Skill 1: style-distiller（写作风格蒸馏器）

> 让 AI 写出来的东西"像你"，而不是"像模板"。

| 维度 | 内容 |
|------|------|
| **解决的问题** | AI 写文章总是"模板味儿"——小红书体、知乎体千篇一律。用户想让 AI 按自己的风格写，但 AI 不认识他。 |
| **目标用户** | 经常写短文（小红书/即刻/朋友圈/微博），想让 AI 辅助创作但不想失去个人风格的写作者 |
| **核心能力** | 喂入喜欢的文章 → 自动蒸馏风格特征 → 按档案生成文章 → 打分反向校准 → 档案持续进化 |
| **技术亮点** | ① 动作级提取（"开头 67% 用对话"而非"文笔好"）② 5 道质量护城河 ③ 7 维自动验证 + 偏离自动修复 |
| **关键数据** | 3 正样本 + 1 反样本 → 95% 风格还原度；3 轮迭代后从 30%→92% 实际匹配 |
| **链接** | [GitHub](https://github.com/MiaIria/style-distiller) · 开源、含完整 README 和安装指南 |

### 🎯 产品设计思路

**核心洞察**：用户说"写得像我"，但 LLM 不理解"我"是谁。fine-tuning 太重，模板化 prompt 太浅。需要一个**轻量但持续进化的"个人写作 LoRA"**——不改模型参数，靠上下文工程 + 检索增强 + 规则蒸馏 + 反馈循环达到"准训练"效果。

**关键决策**：
- **选"风格蒸馏"而非"模板套用"**：从 6 个候选方向中选定，因为用户的核心诉求是"持续进化"而非"快速出稿"
- **短文优先**：聚焦小红书/即刻/朋友圈（100-800 字），长文可用但不优化——先做深一个场景
- **自建反样本机制**：用户不光知道喜欢什么，也知道讨厌什么（"空洞大道理""网络梗堆砌"），反样本让档案有一半是靠"不要什么"来定义的
- **数据与逻辑分离**：工具代码开源（GitHub），个人风格数据永不上传（本地 `~/.claude/styles/`）

### 🏗️ 架构亮点

```
5 个公开 Skill（各 ~120 行入口文件，轻量）
       ↓ 全部引用
1 个共享库 style-lib（13 文件 / ~2500 行，真正的引擎）
       ↓ 读写
1 份本地风格档案（8 维画像 + 正/反样本库 + 草稿 + 备份）
```

**5 道质量保证护城河**（防止"训练 50 篇白费"）：

| # | 护城河 | 作用 |
|---|--------|------|
| 1 | 检索要准 | 从 50 个样本中挑 3-5 个最相关的（主题 + 维度 + 时效 多路召回） |
| 2 | 档案翻译要硬 | 软描述 → 🚨🟡🟢 三层硬约束（如"22 个禁用词，0 容忍"） |
| 3 | Prompt 组装要全 | 嵌入完整原文 + 7 维特征签名，不只是摘要 |
| 4 | 生成后自动验证 | 7 维核对 + 反样本扫描 + 样本特征还原度检查 |
| 5 | 偏离自动修复 | 偏离 10-25% 自动改，>50% 自动重写（最多 2 次） |

### 📊 效果验证

**2026/06/03 全流程测试**：冷启动 → 喂 3 正 + 1 反样本 → 试写 → 评分 → 反向校准，全流程跑通。

**关键发现**——一次"失败"暴露了系统的真实边界：

| 版本 | 评分 | 发现 |
|------|------|------|
| 草稿 #005 | 1-3/10 | **"表面匹配 ≠ 内容准确"**。7 维验证能守住形式约束（禁用词、句长、开头类型），但抓不住"主题理解偏差"。 |
| 草稿 #006 | 4-6/10 | 修复了形式问题，但"中间缺案例支撑"——用户反馈催生了"双案例堆叠"新范式。 |
| 草稿 #007 | 7-8/10 | 3 轮迭代后实际匹配从 30% → 92%，反馈闭环验证有效。 |

> 这个 crash-and-recovery 过程恰恰是产品最值得说的部分：不是一上来就完美，而是有机制能从失败中学习。

## 📖 Skill 2: love-match（情感人格测评）

> "这说的明明就是我啊" —— 让用户在 18 道题后发自内心说出这句话。

| 维度 | 内容 |
|------|------|
| **解决的问题** | MBTI/星座太泛泛，不触及情感领域的深层模式。用户想要"看清自己在恋爱里的样子"，而不是一个笼统的人格标签。 |
| **目标用户** | 对自我认知有需求、正在或准备进入亲密关系的年轻人 |
| **核心能力** | 18 道递进式选择题 → 5 个情感阶段 → 7 维情感分析 → 12 条洞察规则 → 3 维专属标签（64 种组合） |
| **技术亮点** | ① 12 条洞察规则防止 LLM 分析质量波动 ② 3 维标签系统类似 MBTI 但聚焦情感领域 ③ 逐题呈现降低参与门槛 ④ 巴纳姆效应主动规避 |
| **关键数据** | 5 轮设计迭代 / 18 题 4 选项 / 7 维 × 12 规则 × 64 标签组合 / 已部署 H5 在线体验 |
| **链接** | [GitHub](https://github.com/MiaIria/love-persona) · [在线体验](https://love-manual.taro.black) |

### 🎯 产品设计思路

**核心洞察**：市面上的情感测试要么太娱乐化（"测测你的恋爱运势"），要么太学术化（ECR 依恋量表，普通人看不懂）。需要一个**有心理学依据、但呈现方式轻量友好**的中间态产品。

**关键决策**：
- **选择题而非开放式对话**：降低参与门槛，保证题目一致性，让分析结果可复现
- **逐题呈现而非一页展示**：每次只看 1 题，用户不用面对"还有 17 题"的心理压力
- **75% 自我解析 + 25% 理想型画像**：先让用户看清自己，再谈匹配——不做"你适合找什么样的人"的主导叙事
- **12 条规则硬编码**：不让 LLM 自由发挥分析——规则库充当"护栏"，保证每次输出都达到最低质量标准

### 🏗️ 架构亮点

```
问卷模式（默认）              对话模式（可选）
  18 题逐题呈现              开放式情感收集
       ↓                         ↓
  7 维情感评分引擎（加权映射）
       ↓
  12 条规则库检测（跨维度模式匹配）
       ↓
  3 维标签生成（64 种组合）
       ↓
  报告输出：自我解析(75%) + 理想型(25%)
```

**12 条洞察规则**分为 5 类：防御/保护（A）、矛盾/张力（B）、爱的语言（C）、安全感（D）、连接深度（E）。每条规则都检测特定的跨维度答案组合，例如：

> **规则 A1「三重保护壳」**：当 Q7=隐藏情绪 + Q10=压抑情绪 + Q16=高度警觉 → 触发"你习惯把脆弱藏得很深，可能需要很长时间才能信任一个人"的深层解读。

**H5 部署**：分析引擎（`LoveMatchAnalyzer` 类）完全在浏览器端运行，无服务器依赖，所有评分和洞察生成都在本地完成。

### 📊 设计迭代

| 轮次 | 改动 | 原因 |
|------|------|------|
| V1 → V2 | 从 5 题扩展到 18 题 + 5 阶段递进 | 5 题太浅，无法支撑深度分析 |
| V2 → V3 | 删除 6 种"恋爱人格原型"，改为 7 维分析框架 | 人格标签太笼统，维度分析更精准 |
| V3 → V4 | 重写第 1 阶段为"心动原点"（从生活画像改为情感触发） | 用户反馈"前面的题不像在做情感测评" |
| V4 → V5 | 新增 3 维标签 + 12 条洞察规则库 | 维度分析好但缺"一句话总结"，规则库防 LLM 波动 |

## 📖 Skill 3: ai-job-search-assistant（AI 求职助手）

> 从"搜岗位"到"面建议"，5 步流水线完成一次完整的求职调研。

| 维度 | 内容 |
|------|------|
| **解决的问题** | 投简历前最耗时的是调研——搜公司、筛匹配、查背景、改简历、准备面试。每个环节都要在不同平台跳转，信息碎片化严重。 |
| **目标用户** | 正在找 AI/互联网方向工作的求职者（应届生到 3-5 年经验） |
| **核心能力** | 输入目标岗位 + 城市 + 简历 PDF → 自动搜索 50 家公司 → 匹配评分筛选 20-25 家 → 深度调研 15 家 → 输出 8 家优先投递建议 + STAR 简历改写 + 面试预测题 |
| **技术亮点** | ① 5 步流水线每步有独立 LLM 角色定义 ② 100 分匹配量表（硬要求/经验/软匹配三轴）③ 5 维企业深度调研框架 ④ 支持排除已调研公司（增量使用） |
| **关键数据** | 5 步完整流程 / 50→25→15→8 漏斗筛选 / 100 分匹配量表 / 已实际使用 74+ 次 |

### 🎯 产品设计思路

**核心洞察**：求职调研的本质是一个**信息聚合 + 匹配分析**问题。传统做法是手动在不同平台搜索、用 Excel 整理、凭感觉筛选。LLM 在这类"结构化搜索 + 评分 + 报告"任务上天然擅长。

**关键决策**：
- **分步执行而非一步到位**：5 步设计让用户可以在每一步检查结果、调整方向——"50 家里有没有漏掉的？""这 8 家优先级排序合理吗？"
- **量化的匹配量表**：硬要求 30 分 + 经验 40 分 + 软匹配 30 分 = 100 分。避免 LLM 凭"感觉"推荐，强制量化思考。
- **角色化提示词**：每一步给 LLM 不同的角色身份（招聘顾问 → HR 专家 → 企业研究员 → 职业教练 → 报告撰写师），触发更专业的回答模式
- **增量使用设计**：支持传入"已调研公司列表"排除重复，让用户可以在几天内分多次使用而不重复劳动

### 🏗️ 架构亮点

```
Step 1: 搜索筛选（50 家公司，多平台聚合）
    ↓
Step 2: 简历解析 + 匹配评分（50 → 20-25 家，100 分量表）
    ↓
Step 3: 深度调研（15 家，5 维框架：基础/文化/制度/薪酬/AI团队）
    ↓
Step 4: 匹配分析 + 简历优化（8 家优先，STAR 改写 + 面试预测题）
    ↓
Step 5: 最终报告（候选人画像 + 公司列表 + 深度报告 + 投递建议 + 附录）
```

**100 分匹配量表**（用于 Step 2）：

| 维度 | 分值 | 考察内容 |
|------|------|---------|
| 硬要求匹配 | 30 分 | 学历、年限、必需技能 |
| 经验相关性 | 40 分 | 行业、职能、项目经验 |
| 软性匹配 | 30 分 | 公司阶段、技术栈、成长空间 |

评分 ≥ 90 强烈推荐 / 75-89 推荐 / 60-74 可行 / <60 不推荐。

## 🧠 贯穿所有 Skill 的设计理念

回头看这 3 个 Skill，它们不是 3 个独立的玩具——背后有一条一致的产品哲学：

### 1. "AI 不做全部，AI 做加速器"

style-distiller 不替你写，它学你怎么写；love-match 不告诉你该爱谁，它帮你看清自己；job-search 不帮你投简历，它帮你把调研时间从 3 天压缩到 30 分钟。**每个 Skill 的边界都很清晰：AI 做信息处理和模式识别，人做决策。**

### 2. "软描述 → 硬约束"

LLM 最擅长说"文笔细腻"这种漂亮但不可执行的话。我的做法是把它翻译成具体动作：不是"开头要抓人"，而是"67% 的文章用对话开场，33% 用场景锚定"。**让模糊的审美变成可验证的规则**，这是这 3 个 Skill 共有的技术基因。

### 3. "先量化，再优化"

love-match 有 12 条硬编码洞察规则不依赖 LLM 自由发挥，style-distiller 有 7 维自动验证 + 偏离度评分，job-search 有 100 分量表。**不能度量的东西就无法迭代**——每个 Skill 都内置了量化反馈机制。

### 4. "轻量但严肃"

这些都是**纯 prompt 工程**（没有训练模型、没有向量数据库、没有后端服务），但不代表它们是"玩具"。style-distiller 的 v0.1.0 包含了 2,500+ 行经过精心设计的提示词、5 道质量护城河、11 条待修复问题的优先级 backlog。**工程严肃性不取决于用了多少技术栈，取决于设计者多想清楚。**
